# 2. 基本的Linux命令学习

  • 2.1 cd:改变目录(Change directory)
  • 2.2 cd..:返回上一层目录
  • 2.3 pwd:显示当前所在目录
  • 2.4 ls(ll):列出当前目录中的所有文件,只不过ll列出的内容更详细
  • 2.5 touch:新建一个文件 如 touch main.js就在当前目录下新建一个main.js文件
  • 2.6 mkdir:新建一个文件夹(make )
  • 2.7 rm:删除一个文件,rm 文件名就删除对应的文件(remove 移除)
  • 2.8 rm -r:删除一个文件夹,rm 文件夹名就删除对应的文件夹(remove 移除)
  • 2.9 mv 移动文件(move) mv index.js test index.js和test在同一个文件夹下,将index.js移动到test中
  • 2.10 reset 重新初始化终端/清屏
  • 2.11 clear 清屏
  • 2.12 history 查看命令历史
  • 2.13 help 帮助
  • 2.14 exit 退出
  • 2.15 #表示注释

# 6. Git文件操作

  • git add. 将文件添加到暂存区

忽略文件
有时候不想把某些文件纳入版本控制里面,比如数据库文件、临时文件、设计文件等
在主目录下建立“.gitignore”文件，文件有如下规则：
*.txt            #忽略所有.txt结尾的文件
！lib.txt  	    #但是lib.
/temp            #仅忽略项目根目录下的TODO文件，不包括其它目录temp
bulid/           #忽略build/目录下的所有文件

# 7. Git分支

# 列出所有本地分支
$ git branch

# 列出所有远程分支
$ git branch -r

# 列出所有本地分支和远程分支
$ git branch -a

# 新建一个分支，但依然停留在当前分支
$ git branch [branch-name]

# 新建一个分支，并切换到该分支
$ git checkout -b [branch]

# 新建一个分支，指向指定commit
$ git branch [branch] [commit]

# 新建一个分支，与指定的远程分支建立追踪关系
$ git branch --track [branch] [remote-branch]

# 切换到指定分支，并更新工作区
$ git checkout [branch-name]

# 切换到上一个分支
$ git checkout -

# 建立追踪关系，在现有分支与指定的远程分支之间
$ git branch --set-upstream [branch] [remote-branch]

# 合并指定分支到当前分支
$ git merge [branch]

# 选择一个commit，合并进当前分支
$ git cherry-pick [commit]

# 删除分支
$ git branch -d [branch-name]

# 删除远程分支
$ git push origin --delete [branch-name]
$ git branch -dr [remote/branch]
